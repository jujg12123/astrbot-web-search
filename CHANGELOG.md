# 更新日志

## v3.1.5 (2026-06-11)

- 🐛 改进 Bing 中文搜索防拆字策略
  - 不再对整句一刀切加双引号，改为只给中文连续片段加双引号
  - 例如 `原神 最新兑换码` 会变成 `"原神" 最新兑换码`
  - 对 `双花`、`明日方舟 wiki` 这类混合关键词更稳
- 👤 作者字段更新为 `jujg12123`

## v3.1.4 (2026-06-08)

- 🐛 修复 Bing 搜索中文时自动拆单字的问题
  - 新增 CJK 字符检测：Bing 搜索含中文时自动加双引号
  - 例如 `原神` 会以 `"原神"` 进行短语搜索，避免被拆成 `原` + `神` 搜出字典类结果
- 🌐 默认搜索引擎仍为 Bing

## v3.1.3 (2026-06-08)

- 🔧 修复搜索结果被 skills 层二次加工导致信息失真的问题
  - 移除 LLM 工具中的"帮用户总结"指令，搜索结果原样返回给 AI
  - 用户命令输出保留结构化格式但去除累赘的"建议"段落
- 📊 默认结果数从 5 提升到 8
- 🐛 修复 `_conf_schema.json` 使用不支持的 `dropdown` 类型，改为 `string+enum`
- 🌐 默认搜索引擎仍为 Bing

## v3.1.2 (2026-06-08)

- 🌐 Bing 搜索改用中国版 `cn.bing.com`，国内访问更快更稳定
- 📄 新增 CHANGELOG.md

## v3.1.1 (2026-06-08)

- 🔧 修复插件配置不生效的问题（改用 AstrBotConfig 注入）
- 🌐 默认搜索引擎改为 Bing
- 📝 新增 `/websearch changelog` 命令
- 🎨 配置界面增强：每个引擎附带详细描述

## v3.1.0 (2026-06-08)

- 🤖 LLM 函数工具改用 `add_llm_tools()` 显式注册
- 📊 搜索结果双格式：LLM 收到纯文本，用户看到富文本
- 🧹 新增 `terminate()` 清理机制
- 🔤 支持中文别名 `/搜索`

## v3.0.x (2026-06-07)

- 🌐 多引擎支持：搜狗 / Google / Bing
- 📌 Skill 风格结构化输出（核心发现 → 详细信息 → 建议）
- ⚙️ 插件配置界面
- 🤖 LLM 可主动调用搜索函数工具
